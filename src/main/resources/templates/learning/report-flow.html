Report Generation Flow

Step 1: User Request

User navigates to /reports/payroll-summary?country=Kenya&payrollMonth=2025-01

The controller receives the report name and parameters

Step 2: Lookup Report Metadata

ReportBuilderService queries report_sources table:

SELECT * FROM report_sources WHERE rpt_name = 'payroll-summary'

This returns the base query and report description

Step 3: Get Column Definitions

Service queries report_columns table:

SELECT * FROM report_columns WHERE rpt_src_id = ? ORDER BY rpt_col_order

This defines which columns to display and their order

Step 4: Build Dynamic SQL

Service constructs SQL query:

SELECT employee_name, country, department, payroll_month, gross_salary, tax_amount, pension, net_salary
FROM (base_query) base_data
WHERE country = 'Kenya' AND payroll_month = '2025-01'
Step 5: Execute Query

JdbcTemplate executes the query and returns results as List<Map<String, Object>>

Each row is a map of column names to values

Step 6: Render Template

Thymeleaf template receives ReportData object and renders HTML table:

<tr th:each="row : ${report.rows}">
    <td th:each="col : ${report.columns}" th:text="${row.get(col)}"></td>
</tr>
Step 7: Export (Optional)

If user clicks export button:

  • ExportService receives the same ReportData
  • Converts to Excel using Apache POI or CSV using OpenCSV
  • Returns file as download

Key Components

  • ReportController: Handles HTTP requests
  • ReportBuilderService: Orchestrates report generation
  • JdbcTemplate: Executes dynamic SQL queries
  • ExportService: Converts data to Excel/CSV
  • Thymeleaf: Renders HTML views
