Report Metadata System

What is Report Metadata?

Report metadata is a system that defines reports using database tables instead of hardcoded logic. This allows reports to be configured, modified, and created without changing application code.

Metadata Tables

report_sources

Defines the basic information about each report

  • rpt_src_id: Unique identifier
  • rpt_name: Report name (used in URLs)
  • description: Human-readable description
  • base_query: The SQL query that retrieves data
report_columns

Defines which columns to display in the report

  • rpt_src_id: Links to report_sources
  • rpt_col_name: Column name from query
  • rpt_col_alias: Display name for users
  • rpt_col_order: Display order
  • rpt_col_data_type: Data type for formatting
report_parameters

Defines filter parameters for reports

  • rpt_src_id: Links to report_sources
  • parameter_name: Parameter name (e.g., "country")
  • parameter_type: Data type (string, date, etc.)
  • default_value: Default value if not provided

How It Works

  1. User selects a report by name
  2. System looks up report in report_sources
  3. System retrieves columns from report_columns
  4. System builds SQL query using base_query + user parameters
  5. System executes query and formats results
  6. Results are displayed using Thymeleaf template
